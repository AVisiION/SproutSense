## Public Routes

Public pages should be accessible without authentication and include:
- home
- about
- features
- plant library
- demo
- contact

## Layout Strategy

Design goals:
- visual consistency with brand identity
- clear navigation hierarchy
- responsive behavior across desktop and mobile
- clean transition path from public views to auth flows

## Content Strategy

Public content should:
- explain platform value quickly
- show practical capabilities
- include clear calls-to-action
- avoid exposing sensitive system internals

## Homepage Design Guidance

A high-quality homepage should include:
- concise hero message
- problem and solution framing
- feature highlights
- credibility elements and project context
- clear action buttons

## Quality Checklist

- Routes are accessible as guest
- Navigation order is consistent
- Content is readable on mobile
- Links and calls-to-action function correctly
